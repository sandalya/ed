# Ed QA — Словник команд для роботи з AI-оркестратором

## Контекст

Цей документ — практичний словник для Сашка. Показує 15 прикладів спілкування з AI (Kit/Sonnet/Opus) щодо управління автотестами Ed'а. Від найпростіших до найскладніших. Кожен приклад показує:
- **Що каже Сашок** — природна мова, як він реально пише
- **Що чує AI** — як AI інтерпретує команду технічно
- **Що AI робить** — конкретні дії на Pi5

Сашок не пише JSON і не запам'ятовує команди CLI. Він описує що хоче, AI виконує. Ці приклади — не жорсткий синтаксис, а орієнтир щоб обидві сторони розуміли одне одного.

## Рівень 1 — Базові команди (дивлюсь, що є)

**Сашок:** "Покажи блоки"


**AI відповідає:**

```
  01_smoke       — 3 кейси
  02_pricing     — 5 кейсів
  03_catalog     — 4 кейси

В архіві нічого.
```

## Рівень 2 — Додавання і редагування

### Приклад 4: Додати простий кейс

**Сашок:** "Додай у pricing кейс про обручки з гравіюванням"

**AI чує:** Створити новий single-turn кейс у `02_pricing.json`. AI сам генерує id, message, tags, expected_behavior за контекстом блоку (pricing = ціни, InSilver = ювелірка).

**AI робить:**
```
1. Генерує кейс:
   id: price_ring_engraving
   message: "Скільки коштує обручка з гравіюванням?"
   expected_behavior: should_respond_in_ukrainian, should_show_price, should_explain_engraving_options
   expand: 3

2. Показує Сашку перед збереженням
3. Чекає ОК
4. SSH → пише в 02_pricing.json
5. Валідує JSON
```

**AI відповідає:**
```
Додаю в pricing:
  price_ring_engraving — "Скільки коштує обручка з гравіюванням?"
  expand: 3, очікування: ціна + опції гравіювання
  
Зберігаю? (так/ні)
```

## Рівень 5 — Складні операції

1. Зрозуміти контекст (фото-запити — це відома проблема чи нова?)
2. Створити adhoc кейси для цієї ситуації
3. Прогнати і знайти проблему
4. Запропонувати фікс
5. Верифікувати після фіксу

**AI робить (покроково, з Сашком):**

**Крок 1 — Дослідження:**
```
"Фото-запити зараз не тестуються — у жодному блоці нема кейсів 
з посиланням на фото. Direct transport фото не підтримує, 
тільки Telegram. Це нова категорія тестів.

Але текстову частину можемо перевірити зараз.
Додаю в adhoc 3 варіації: 
  'хочу сережки як на фото'
  'подивіться на фото, хочу такі ж'
  'ось фото, зробіть схоже'
  
Прогоню через direct без фото — подивимось як бот реагує

```

**Крок 2 — Прогін:**

**Крок 3 — Пропозиція:**

## Підсумок: що запам'ятати

Сашок не пише JSON, не запам'ятовує CLI-команди, не думає про формати. Він думає про продукт:
- "Влад скаржиться на X" → AI розслідує
- "Додай тести для Y" → AI створює
- "Прогони Z" → AI запускає
- "Що впало?" → AI пояснює
- "Перенеси, архівуй, поверни" → AI організовує

AI тримає порядок у файлах, валідує JSON, показує diff, робить backup. Сашок керує на рівні намірів.
